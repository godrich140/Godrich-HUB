测试入口：http://localhost:5173/

前端已切换为简约浅色工具界面，主要页面包括：

- 工作台
- 历史单据
- 统计查询
- 客户通讯录

最近一次构建验证已通过：

```bash
cd frontend
npm run build
```

本地开发服务启动命令：

```bash
cd frontend
npm run dev -- --port 5173
```

## 测试重点

### 1. 工作台

重点验证：

- 页面默认进入“工作台”。
- 左侧导航、顶部按钮、汇总卡片、基础信息区、明细录入表格、右侧照片识别和付款信息都能正常显示。
- 照片识别初始状态应为“未识别”，进度为 `0%`。
- 点击“上传手写照片”选择图片后，状态应先变为“识别中”，进度为 `68%`，随后变为“识别成功”，进度为 `100%`。
- 拖拽图片到照片识别区域也应触发同样流程。
- 明细录入表格可编辑文本、数字、单位、箱规等字段。
- “新增一行”可以追加新明细行。
- 勾选连续多行后，“合并选中行”应合并图片、数量/箱、箱数、毛重/箱、体积、总毛重、箱子规格、总体积等可合并列。
- “取消合并”应取消选中行的合并状态。
- “清空明细录入”会弹出确认框，确认后保留一条空白行。
- “弹出录入”会打开可调整尺寸的明细录入弹窗。

注意：

- “保存草稿”“预览表格”“生成 Excel”目前是展示按钮，尚未接入真实导出或保存逻辑。
- OCR 识别是前端模拟流程，不是真实识别服务。

### 2. 历史单据

重点验证：

- 点击左侧“历史单据”可进入历史装箱单页面。
- 顶部汇总卡片显示匹配单据、明细行、总金额、总箱数。
- 可通过“单据 / 客户”关键词过滤历史单据。
- 可通过日期过滤历史单据。
- 可通过状态下拉过滤历史单据。
- 左侧历史单据卡片可点击选中。
- 右侧预览区域应随选中单据切换。
- 无匹配结果时应显示空状态提示。

注意：

- “批量导出”“新建装箱单”“预览”“生成 Excel”目前是展示按钮，尚未接入实际业务动作。

### 3. 统计查询

重点验证：

- 点击左侧“统计查询”可进入统计页面。
- 起始日期、终止日期、产品关键词可过滤明细。
- 客户复选项可单选、多选、全选/取消全选。
- 当未选择任何客户时，应显示空状态提示。
- 有匹配数据时，应显示查询明细表格和底部汇总卡片。

### 4. 客户通讯录

重点验证：

- 点击左侧“客户通讯录”可进入客户管理页面。
- 可输入客户名称、联系人、电话、邮箱、地址。
- 点击“保存客户”后，新客户应出现在客户列表。
- 点击客户列表项，应把客户信息回填到表单。
- 搜索框可按客户名称、联系人、电话过滤客户。
- 工作台的客户选择弹窗可选择已有客户，也可新建客户。

## 建议浏览器测试用例

### 用例 A：工作台基础录入

1. 打开 `http://localhost:5173/`。
2. 确认页面为浅色后台工具风格。
3. 在客户输入框填写或选择客户。
4. 修改日期、单号、状态。
5. 修改第一行明细的品名、数量、单价、总价。
6. 点击“新增一行”，确认表格增加新行。
7. 在新行中录入任意明细。

预期结果：输入内容保留，汇总卡片随数量、金额、箱数变化。

### 用例 B：照片识别流程

1. 刷新页面。
2. 确认照片识别为“未识别 0%”。
3. 上传一张图片。
4. 观察状态变化。

预期结果：状态从“未识别 0%”变为“识别中 68%”，随后变为“识别成功 100%”；第一行空字段会补入模拟 OCR 内容。

### 用例 C：合并明细行

1. 勾选连续两行或多行。
2. 点击“合并选中行”。
3. 检查箱规相关列是否合并显示。
4. 勾选合并行后点击“取消合并”。

预期结果：连续行可以合并；取消合并后恢复独立行。

### 用例 D：历史单据筛选与预览

1. 进入“历史单据”。
2. 输入历史单据编号或客户关键词。
3. 修改日期或状态筛选。
4. 点击左侧不同历史单据卡片。

预期结果：列表和汇总同步过滤；右侧预览随选中单据变化。

### 用例 E：客户通讯录

1. 进入“客户通讯录”。
2. 输入一个新客户完整信息。
3. 点击“保存客户”。
4. 使用搜索框搜索该客户。
5. 回到工作台，打开客户选择弹窗，选择该客户。

预期结果：客户成功保存、可搜索、可在工作台选择。

## 视觉检查点

- 页面整体应为简约浅色，不应残留黑耀紫主题。
- 侧栏为白底，选中导航为浅蓝高亮。
- 表格表头为浅蓝色，行有斑马纹和 hover 高亮。
- 输入框边框清晰，聚焦时有蓝色外发光。
- 卡片圆角统一为 8px。
- 文字不应溢出按钮、卡片或表格单元。
- 小屏宽度下侧栏应变为顶部横向导航，主要布局应单列堆叠。

## 已知限制

- 当前无后端接口联调，数据为前端内存状态和种子数据。
- 刷新页面后新增客户、录入明细不会持久化。
- Excel 生成、预览、保存草稿、批量导出等按钮尚未实现真实业务动作。
- OCR 仅模拟状态和回填，不进行真实图片识别。
- 图片预览使用浏览器 `blob:` 地址，刷新后会丢失。
